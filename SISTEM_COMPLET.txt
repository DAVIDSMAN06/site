🔄 HOW IT WORKS - STEP BY STEP
════════════════════════════════

SCENARIO: User asks "Ce este gândirea?"

[1] USER ON SITE
    └─ Opens index.html
    └─ Navigates to "Asistent AI"
    └─ Types question: "Ce este gândirea?"
    └─ Clicks: "Trimite"

[2] FRONTEND JAVASCRIPT
    └─ Validates question format
    └─ Shows typing indicator
    └─ Sends HTTP POST to: http://localhost:3000/api/chat
    └─ Body: {question: "Ce este gândirea?"}

[3] NETWORK LAYER
    └─ Request travels over HTTP
    └─ Arrives at backend on port 3000

[4] BACKEND - SERVER.JS
    └─ Receives POST request
    └─ Extracts question
    └─ Checks if question is in course scope
    └─ Uses keyword matching (70+ terms)
    └─ Logic: Has "gândire", "think", "concept"? → OK
    └─ If NO keywords: Return refusal

[5] BACKEND - VALIDATION OK
    └─ Question is course-related
    └─ Prepares OpenAI API call
    └─ Includes system prompt (COURSE_CONTEXT)
    └─ Sets constraints (max_tokens: 500)

[6] BACKEND CALLS OPENAI
    └─ Uses API key from .env
    └─ Sends request over HTTPS to OpenAI servers
    └─ Model: gpt-3.5-turbo
    └─ Waits for response (usually 1-3 seconds)

[7] OPENAI RESPONDS
    └─ Generates answer based on:
       - System prompt (course scope)
       - User's question
       - Model's training
    └─ Returns: ~100-400 words

[8] BACKEND PROCESSES RESPONSE
    └─ Receives from OpenAI
    └─ Formats response
    └─ Returns to frontend:
       {success: true, response: "Gândirea este..."}

[9] NETWORK LAYER
    └─ Response travels back over HTTP

[10] FRONTEND RECEIVES
     └─ Removes typing indicator
     └─ Displays response in chat
     └─ Formats with styling

[11] USER SEES ANSWER! 🎉
     └─ Beautiful response
     └─ Can ask more questions
     └─ Can scroll chat history

🔐 SECURITY MODEL
═════════════════════════════

LAYER 1: Secrets Management
  ├─ API Key stored in .env (NOT in code)
  ├─ .env added to .gitignore (NOT on GitHub)
  ├─ Only backend reads .env (frontend never sees it)
  └─ If leaked, can be revoked on OpenAI platform

LAYER 2: Domain Restriction
  ├─ Server validates question against keywords (70+ terms)
  ├─ System prompt defines scope (COURSE_CONTEXT)
  ├─ OFF-TOPIC questions are REFUSED at server level
  ├─ Costs ZERO $ to refuse (saves money)
  └─ Frontend cannot bypass (JavaScript on client)

LAYER 3: API Protection
  ├─ Direct API calls ONLY from backend
  ├─ Frontend CANNOT call OpenAI directly
  ├─ API key never exposed to frontend
  ├─ All requests go through validated endpoint
  └─ CORS configured for localhost only

LAYER 4: Error Handling
  ├─ 401 Errors (invalid key) → User-friendly message
  ├─ 429 Errors (rate limit) → Suggests waiting
  ├─ Network errors → Fallback offline mode
  ├─ Never reveals sensitive info in errors
  └─ Logs key details for debugging
