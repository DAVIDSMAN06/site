❓ FREQUENTLY ASKED QUESTIONS
══════════════════════════════════


1. CE ESTE ASISTENTUL AI?
────────────────────────

R: E un chat bot alimentat de OpenAI care:
   - Răspunde la orice întrebare despre curs
   - Refuză să vorbească despre alte subiecte
   - Folosește AI real (nu răspunsuri hardcodate)
   - Funcționează instant (1-3 secunde)


2. E GRATUIT?
──────────────

R: NU. Fiecare întrebare costă bani (OpenAI taxes):
   - Mediu: ~0.001-0.01$ per întrebare
   - 100 întrebări ≈ 0.10-1.00$
   - 1000 întrebări ≈ 1.00-10.00$

   IMPORTANT: Setează usage limit pe platform.openai.com
   pentru a preveni suprize


3. CUM OBȚIN API KEY?
──────────────────────

R: 4 Pași:
   1. Mergi pe https://platform.openai.com/api-keys
   2. Conectează-te cu cont Gmail/Microsoft
   3. Apasă "Create new secret key"
   4. Copie cheia (va arăta: sk-proj-ABC123...)
   
   ⚠️ Salvează-o SIGUR! Nu o poți vedea din nou!


4. CE FACI CU CHEIA?
─────────────────────

R: O pui în fișierul .env:
   
   OPENAI_API_KEY=sk-proj-YOUR_KEY_HERE
   
   Fișierul .env e ignorat de Git (privat!)


5. UNDE E .env?
──────────────────

R: În rădăcina proiectului:
   C:\Users\DAVIDSMEN\Desktop\sitebun1\.env
   
   (Lângă server.js, package.json, etc)


6. TREBUIE SĂ MĂ LOGIN ERGO?
───────────────────────────────

R: NU! Cheia din .env e suficientă.
   Frontend-ul nu vede cheia (e pe server).


7. SERVERUL NU PORNESTE!
──────────────────────────

R: Cause-uri posibile:
   ❌ Port 3000 e ocupat
      → Schimbă PORT în .env (3001, 3002, etc)
   
   ❌ npm install nu a rulat
      → npm install (din folder)
   
   ❌ Node.js nu e instalat
      → https://nodejs.org/ (LTS)
   
   ❌ .env nu e setat
      → Editează .env cu cheia


8. AI NE RASPUNDE?
─────────────────────

R: Verifică:
   [  ] npm start rulează (PowerShell)
   [  ] Linia "🤖 AI Server funcționează..." apare
   [  ] Frontend shows "🤖 AI Real Conectat!"
   [  ] Ai internet
   [  ] API Key e valid (npm test)


9. FRONTEND SPUNE "BACKEND NU E PORNIT"
────────────────────────────────────────

R: Opriri și restartează:
   1. Închide PowerShell (npm start)
   2. Deschide PowerShell nouă
   3. cd C:\Users\DAVIDSMEN\Desktop\sitebun1
   4. npm start
   
   Ar trebui să fie verde și priet!


10. "401 UNAUTHORIZED" ERROR?
──────────────────────────────

R: API Key e invalid:
   1. Mergi pe platform.openai.com/api-keys
   2. Generează NOUĂ cheie (stara poate fi expirată)
   3. Copie și pune în .env
   4. npm start din nou
   5. npm test (verifica)


11. "429 TOO MANY REQUESTS"?
──────────────────────────────

R: Rămâi fără rate limit:
   1. Stai 1 minut
   2. Setează usage limit pe OpenAI
   3. Poți reduce întrebări/minut


12. "ENOTFOUND openai"?
─────────────────────────

R: Probleme de internet/DNS:
   1. Verifică conexiunea
   2. Ping google.com (PowerShell):
      ping google.com
   3. Dacă merge, restart npm start
   4. Dacă nu merge, internet e down


13. "Cannot find module 'express'"?
──────────────────────────────────────

R: npm install nu a mers:
   1. Șterge folder node_modules (dacă există)
   2. Șterge package-lock.json
   3. npm install
   4. npm start


14. SITE MAN FUNCȚIONEAZĂ FĂRĂ INTERNET?
───────────────────────────────────────────

R: Parțial:
   ✅ Lecții: DA (offline)
   ✅ Quiz: DA (offline)
   ❌ AI Chat: NU (trebuie internet + server)


15. POT SĂ PUN AI-UL LIVE PE INTERNET?

R: DA, dar trebuie:

   - API Key protejat cu variabile de mediu

   (Detalii în AI_DOCUMENTATION.txt)

17. CUM E SIGUR CHEIA?
────────────────────────

R: Sigură dacă:
   ✅ .env e în .gitignore (nu se commitează)
   ✅ .env e pe serverul tău (NU pe client)

   ✅ Cheia se schimbă dacă e compromisă
   
   Nesigur dacă:
   ❌ .env e pe GitHub
   ❌ Cheia e hardcoded în JavaScript
   ❌ Oricine o poate vedea

🔧 TROUBLESHOOTING AVANSAT
═══════════════════════════════


PROBLEMA: "Express server crashes"
──────────────────────────────────

Diagnoza:
  1. Deschide PowerShell
  2. npm start
  3. Notează error-ul exact
  4. Dacă e "EADDRINUSE":
     → Port e ocupat
     → Schimbă PORT în .env
  
  Soluție:
  PORT=3001


PROBLEMA: "OpenAI API key valid dar nu merge"
──────────────────────────────────────────────

Diagnoza:
  1. npm test (testează direct)
  2. Check dacă ți-a expirat creditul
  3. Mergi pe platform.openai.com:
     - Account usage
     - Billing
     - Check credit balance

Soluție:
  - Adaugă credit
  - Generiază NOUĂ cheie
  - Testează din nou


PROBLEMA: "npm install merge dar npm start e lent"
────────────────────────────────────────────────────

Cauze:
  - Virusul scan
  - HDD lent
  - Firewall blochează

Soluție:
  npm install --no-save


PROBLEMA: "Frontend e ok, dar AI nu răspunde"
───────────────────────────────────────────────

Debug steps:
  1. Deschide DevTools (F12)
  2. Mergi la Console
  3. Put o întrebare
  4. Notează error-ul
  
  Cauze comune:
  - CORS blochează
  - Serverul nu rulează
  - Port greșit
  - API Key invalid

Soluție:
  1. Verific npm start
  2. npm test
  3. Restartează


PROBLEMA: "API response lent sau timeout"
───────────────────────────────────────────

Cauze:
  - OpenAI server e ocupat
  - Conexiune lentă
  - Request prea lung

Soluție:
  1. Stai 1 minut
  2. Încearcă din nou
  3. Pune o întrebare mai scurtă

💡 PRO TIPS
═════════════


TIP 1: Mod Development
──────────────────────
npm run dev
(Auto-reloads la orice schimbare în server.js)


TIP 2: Check Logs
──────────────────
Privete output din PowerShell:
- "Request from client" = user a pus întrebare
- "Calling OpenAI" = server contactează API
- "Response sent" = răspuns trimis la client


TIP 3: Rate Limiting
─────────────────────
OpenAI limit: 3-4 requests/minute pentru free
Soluție: Stai între întrebări


TIP 4: Optimize Responses
──────────────────────────
Editează max_tokens în server.js:
max_tokens: 300 (scurt + ieftin)
max_tokens: 500 (normal - DEFAULT)
max_tokens: 1000 (lung + scump)


TIP 5: Offline Fallback
────────────────────────
Dacă serverul e down, frontend-ul folosește
răspunsuri offline (limitate) automat.


TIP 6: Test Endpoint Direct
─────────────────────────────
Poți testa API cu curl (PowerShell):

$json = @{
    question = "Ce este gândirea?"
} | ConvertTo-Json

$response = Invoke-WebRequest -Uri "http://localhost:3000/api/chat" `
    -Method POST `
    -Body $json `
    -ContentType "application/json"

$response.Content
