📊 FLUX COMPLET SISTEM
═══════════════════════════════

1. USER OPENS index.html
   ↓
2. FRONTEND (HTML/JS)
   - 5 Lecții cu conținut
   - Quiz 10 întrebări
   - AI Chat Interface
   ↓
3. USER SELECTS "ASISTENT AI"
   ↓
4. FRONTEND CHECKS
   - Backend e pornit? (GET /api/health)
   - Dacă DA: Connected state ✅
   - Dacă NU: Offline mode (limited)
   ↓
5. USER PUTS QUESTION
   ↓
6. FRONTEND SENDS (Fetch API)
   POST http://localhost:3000/api/chat
   {question: "Ce este gândirea?"}
   ↓
7. BACKEND VALIDATES (server.js)
   - E întrebarea din domeniu? (keyword matching)
   - Format valid?
   - Dacă OK: Continue
   - Dacă NU: Return "Pot răspunde DOAR..."
   ↓
8. BACKEND CALLS OPENAI
   - Folosește API Key din .env
   - Model: gpt-3.5-turbo
   - Max tokens: 500
   ↓
9. OPENAI RESPONDS
   - ~1-3 secunde
   - Textul răspunsului
   ↓
10. BACKEND RETURNS RESPONSE
    {success: true, response: "..."}
    ↓
11. FRONTEND DISPLAYS
    - In chat window
    - Formatted nice
    ↓
12. USER SEES ANSWER 🎉

🔐 SECURITY LAYERS
═══════════════════════════════

LAYER 1: Environment Protection
  - API Key in .env (NOT in code)
  - .env in .gitignore (NOT on GitHub)

LAYER 2: Server-Side Validation
  - Keyword matching (70+ terms)
  - COURSE_CONTEXT system prompt
  - Refuse non-course questions

LAYER 3: API Protection
  - Only backend calls OpenAI (not frontend)
  - API Key never sent to client
  - CORS configured for localhost

LAYER 4: Error Handling
  - 401: Invalid key → Error message
  - 429: Rate limit → Wait message
  - Network error → Fallback offline mode
